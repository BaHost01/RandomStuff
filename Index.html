OVERVIEW

What is XMCLib?

XMCLib is a full-featured Roblox UI framework for building rich, themeable interfaces in both executor and LocalScript contexts. It provides windows, tabs, sections, and a complete set of interactive elements — all with consistent theming, smooth animations, and clean memory management.

Key Features

  • ✓ Memory-safe: all global connections are tracked and disconnected on destroy
  • ✓ Touch / mobile support on Slider, Dropdown, Drag
  • ✓ Dynamic ZIndex — no conflicts between multiple windows
  • ✓ 4 built-in themes (Dark, Purple, Red, Green) + custom theme support
  • ✓ Config system with JSON save/load
  • ✓ Custom signal registry

Load XMCLib via loadstring from an executor, or require it as a ModuleScript.

-- Load and initialise XMCLib
local XMCLib = loadstring(game:HttpGet("https://your-host/XMCLib.lua"))()

local lib  = require(XMCLib)
local win  = lib:CreateWindow({ Title = "My Script", Width = 620, Height = 440 })
local tab  = win:CreateTab("Main", "⚙")

sec:CreateToggle({
    Name     = "Enable Feature",
    Default  = false,
    Flag     = "featureOn",
    Callback = function(state)
        print("Feature:", state)
    end,
})

lib:BindToggleKey(Enum.KeyCode.RightShift)

CHANGELOG

Changelog

v2.1.0

  • FIXMemory leaks — all UserInputService global connections are now stored in element._connections and cleanly Disconnect()ed when :Destroy() is called on Slider, Dropdown, and Keybind.
  • FIXMobile / Touch support — Slider, Dropdown, and the Drag module now handle Enum.UserInputType.Touch alongside mouse events everywhere.
  • FIXDynamic ZIndex — panels and notifications no longer use hardcoded ZIndex values. Every window is assigned a base ZIndex tier; all children and floating panels are computed relative to it, preventing conflicts.
CORE

XMCLib

The root object. Created via XMCLib.new().

XMCLib.new () → XMCLib

Creates a new XMCLib instance. Mounts a ScreenGui (tries gethui(), then CoreGui, then PlayerGui) and initialises the Dark theme.

:CreateWindow (opts: table | string) → Window

Creates a draggable, animated window with sidebar, tabs, and window controls.

FieldTypeDescription
TitlestringWindow title shown in the sidebar logo area.
Width optnumberWindow width in pixels. Default: 600.
Height optnumberWindow height in pixels. Default: 430.
Position optUDim2Initial position. Default: centered.
:BindToggleKey (key: Enum.KeyCode) → void

Binds a keyboard shortcut to toggle visibility of all windows. Typical usage: Enum.KeyCode.RightShift.

:SetVisible / :ToggleVisible (state: boolean?) → void

Show or hide all windows. ToggleVisible flips the current state.

:Notify (opts: table) → Frame

See the Notifications section.

:Destroy () → void

Destroys the entire ScreenGui, removing all UI from the game.

⚠ NOTE Themes apply at creation time. Changing the theme after windows are created does not live-recolor existing elements — destroy and recreate windows to apply a new theme.
CONFIG

Config System

Each element that accepts a Flag option automatically registers and syncs its state with the Config module. You can save/load to disk (executor only).

-- Register a flag manually
lib.Config:Register("myFlag", false)

-- Read / write a flag
lib.Config:Set("myFlag", true)
local val = lib.Config:Get("myFlag") -- true

-- Persist to file (executor only)
lib.Config:Save("profile1")  -- writes XMCLib_profile1.json
lib.Config:Load("profile1")  -- restores all flag values
lua
MethodReturnsDescription
:Register(flag, default)voidRegisters a flag with a default value. No-op if already registered.
:Set(flag, val)voidSets a flag value.
:Get(flag)anyReturns current flag value.
:Save(name?)bool, msgSaves all flags to XMCLib_{name}.json.
:Load(name?)bool, msgLoads flags from XMCLib_{name}.json.

Notifications

Stacked, auto-dismissing toast notifications with a progress bar. Four types: info, success, warning, error.

    Title    = "Success",
    Message  = "Config saved!",
    Type     = "success",  -- "info" | "success" | "warning" | "error"
    Duration = 4,           -- seconds
})
lua
FieldTypeDescription
TitlestringBold heading text.

Type optstring"info" · "success" · "warning" · "error". Default: "info".
Duration optnumberAuto-dismiss time in seconds. Default: 4.
COMPONENT

Window

A floating, draggable frame with a sidebar for tab navigation and a top bar with minimize / close controls. Returns a Window object.

local win = lib:CreateWindow({
    Title    = "My UI",
    Width    = 620,
    Height   = 440,
    Position = UDim2.new(0.5, -310, 0.5, -220),
})
lua
ℹ ZIndex tiers Each window claims a unique ZIndex tier (multiples of 20). Tier 1 = ZIndex 10–29, Tier 2 = 30–49, etc. Children are offset inside the tier, so multiple windows never conflict.

ELEMENT

Dropdown

A floating panel anchored below the button. Closes when clicking outside. Supports touch on mobile.

local dd = sec:CreateDropdown({
    Name    = "Team",
    Items   = { "Red", "Blue", "Green" },
    Default = "Red",
    Flag    = "team",
    Callback= function(val) print(val) end,
})

dd:Set("Blue")
dd:Get()                   -- → "Blue"
dd:AddItem("Yellow")       -- appends
dd:SetItems({ "A", "B" }) -- replaces all items
dd:Destroy()               -- disconnects UIS + destroys floating panel

ELEMENT

Keybind

Click to enter listen mode, then press any key to bind it. The callback fires every time the bound key is pressed.

local kb = sec:CreateKeybind({
    Name    = "Toggle Fly",
    Default = Enum.KeyCode.F,
    Flag    = "flyKey",
    Callback= function(keyCode)
        print("Key pressed:", keyCode.Name)
    end,
})

kb:Get()                      -- → Enum.KeyCode
kb:Set(Enum.KeyCode.G)
kb:Destroy()                   -- disconnects all UIS listeners
lua
⚠ Two connection types The keybind element uses two kinds of UIS connections: a temporary listen connection (active only while awaiting input) and a permanent press connection. Both are cleaned up by :Destroy().

INTERNAL

Custom Signals

A lightweight signal registry not tied to Roblox Instances.

-- Create a signal
CustomSignal("onDamage", nil, "newsignal")

-- Subscribe
CustomSignal("onDamage", function(params)
    print("Damage:", params.amount)
end, "onsignal")

-- Fire
CustomSignal("onDamage", { amount = 50 }, "firesignal")
lua
